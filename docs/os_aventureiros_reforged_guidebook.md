# Página 05 - Guia Refatorado

## Twilight Edition - 1.21.1 / NeoForge

## 1. Espírito da campanha

Os Aventureiros Reforged - Twilight Edition é uma campanha Vanilla+ de aventura, exploração, estruturas e dimensões. Esta versão substitui a base antiga 1.12.2 por um pack atual em Minecraft 1.21.1 com NeoForge, mantendo o espírito de grupo: base compartilhada, expedições, loot, mapas, histórias e grandes jornadas.

**Regra de ouro:** grandes descobertas, primeiras entradas em dimensões, torres importantes e chefes devem acontecer com o máximo possível do grupo online.

## Setup base do modpack

| Item | Valor recomendado |
|---|---|
| Minecraft | 1.21.1 |
| Mod loader | NeoForge 21.1.209 |
| Java | Java 21 |
| RAM cliente | 4 GB mínimo; 6 GB recomendado; 6-8 GB se usar shaders |
| RAM servidor 3 jogadores | 4-6 GB, ajustando conforme render distance, estruturas e exploração simultânea |
| Tipo de mundo | Mundo padrão com Terralith ativo; não usar configuração antiga de Biomes O Plenty |
| Launcher | CurseForge App ou Prism Launcher importando o ZIP do modpack |

O ZIP do modpack funciona como manifesto: ele aponta para os arquivos oficiais dos mods, permitindo que o launcher baixe tudo da fonte correta. Isso reduz conflito de licença, facilita atualização e deixa a instalação mais segura para o grupo.

# Página 07 - Regras da Mesa

## Aventura compartilhada acima de corrida

- KeepInventory desligado, salvo decisão explícita do grupo para uma temporada mais casual.
- Battle Towers, dungeons importantes e chefes dimensionais só contam como conquistados se pelo menos dois jogadores participarem.
- Todo lugar marcante ganha nome: torre, vila, bioma raro, portal, caverna, waystone ou ponto de morte memorável.
- O Lootr garante baús por jogador, mas loot raro e troféus devem ser discutidos no salão da base para manter o espírito de equipe.
- Sem teleport de comando, exceto para corrigir bug, travamento, queda injusta ou problema técnico.
- Antes de abrir uma nova dimensão, o grupo precisa cumprir uma meta mínima de preparação: comida, camas, mochilas, blocos, waystone e rota marcada.

## Funções sugeridas

| Função | Responsabilidade principal |
|---|---|
| Explorador | Usa JourneyMap, Nature's Compass e Explorer's Compass para mapear biomas, estruturas e rotas seguras. |
| Mineiro/Ferreiro | Coleta ferro, carvão, diamante e recursos estratégicos; prepara ferramentas e kits de expedição. |
| Construtor/Guardião | Organiza base, armazéns, mural de missões, fazenda, camas, oficina e defesas. |
| Cartógrafo | Registra waypoints, nomes de locais, rotas de viagem e pontos de retorno. |
| Curador de loot | Ajuda a separar loot de expedição, troféus, recursos comuns e itens raros do grupo. |

# Página 09 - Visual e Performance

## Shaders modernos sem OptiFine

## 8. Shaders e texturas

Nesta versão, o caminho correto para shaders é Iris Shaders + Sodium. Não use OptiFine como base do pack: ele é desnecessário aqui e costuma gerar conflito com mods modernos.

| Perfil visual | Sugestão | Quando usar |
|---|---|---|
| Estável | Sem shader; resource pack Vanilla ou Faithful/estilo limpo | Melhor padrão para todos entrarem e testarem sem conflito. |
| Leve | Shader leve via Iris | PCs modestos; manter render distance menor. |
| Cinemático | Complementary Reimagined ou shader similar compatível com Iris | Fotos, pôr do sol, base e expedições com visual forte. |
| Servidor | Sem shader no servidor | Shaders rodam no cliente; o servidor precisa focar em RAM, CPU e distância de simulação. |

## 9. Checklist de instalação

- [ ] Importar o ZIP no CurseForge App ou Prism Launcher.
- [ ] Garantir Java 21 instalado/selecionado.
- [ ] Alocar 4-6 GB de RAM no cliente; subir para 6-8 GB se usar shader pesado.
- [ ] Abrir singleplayer uma vez antes de entrar no servidor.
- [ ] Confirmar que todos estão na mesma versão do pack.
- [ ] Adicionar shader apenas depois que o pack abrir sem erro.
- [ ] No servidor, ajustar view distance e simulation distance antes de aumentar mods ou estruturas.

# Página 10 - Informações Técnicas

## O que ainda não devemos chutar

## 10. Antes da expansão

Para a futura Expansão de Combate e Armaduras, o handoff package deve evitar IDs inventados. Antes de gerar o addon completo, precisamos extrair ou confirmar dados reais do ambiente do Codex/VPS.

| Informação | Por que é necessária |
|---|---|
| IDs de mods carregados | Confirmar mod ids reais: `aether`, `twilightforest`, `lootr`, `waystones`, `bettercombat`, `spartan` etc. |
| IDs de biomas e dimensões | Evitar worldgen quebrado em Aether/Twilight/Terralith. |
| IDs de estruturas e loot tables | Injetar loot sem apontar para tabelas inexistentes. |
| Versões exatas de Better Combat e Spartan | Gerar compatibilidade opcional sem depender de API errada. |
| Formato real de weapon attributes | Better Combat pode usar caminho/formato específico na versão instalada. |
| Logs de boot client/server | Detectar missing registries, mixin warnings e conflitos antes de empacotar. |
| Dump de registries/datagen | Base segura para tags, recipes, loot modifiers e worldgen. |

A ordem correta é: primeiro este guia refatorado; depois o documento da expansão; depois imagens/PDF da expansão; por fim o handoff package técnico com scaffold do addon.
